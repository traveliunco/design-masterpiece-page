# سير عمل تطوير ترافليون

## قواعد مهمة

1. **لا تشغل التطبيق أبداً** - لا تستخدم `npm run dev` أو أي أمر تشغيل
2. **لا تستخدم npm run build** إلا إذا طلب المستخدم صراحة
3. **التعديلات على الملفات فقط** - قم بالتعديلات مباشرة على الكود

## خطوات التطوير

1. **فحص المشروع** - استخدم `view_file` و `list_dir` لفهم الهيكل
2. **تحري الأخطاء** - استخدم `grep_search` للبحث عن مشاكل
3. **إجراء التعديلات** - استخدم `replace_file_content` أو `multi_replace_file_content`
4. **توثيق التغييرات** - حدّث ملفات التطوير

## ملفات التطوير المهمة

- `DEVELOPMENT_COMPLETE.md` - الوثيقة الشاملة
- `DEVELOPMENT.md` - معلومات المشروع الأساسية
- `DEVELOPMENT_LOG.md` - سجل التطوير التفصيلي

## الأوامر المسموحة

```bash
# فحص الأخطاء فقط (لا تشغل)
npx tsc --noEmit

# فحص ESLint فقط (لا تشغل)
npm run lint
```

## الأوامر الممنوعة

```bash
# ❌ ممنوع
npm run dev
npm start
npm run preview
```
