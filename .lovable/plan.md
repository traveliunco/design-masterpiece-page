

# تحليل وإصلاح صفحات لوحة التحكم

## نتائج الفحص الشامل

بعد فحص جميع صفحات القائمة الجانبية (28 صفحة admin)، وجدت التالي:

### الصفحات المكتملة والعاملة (20 صفحة)
لوحة التحكم، الحجوزات، الوجهات (+ إضافة/تعديل)، البرامج (+ إضافة/تعديل)، العروض (+ إضافة/تعديل)، البكجات الجاهزة، الفنادق، الرحلات، الرسائل، المستخدمين، الدول/المدن/نقل البيانات، الخدمات (+ إضافة/تعديل)، شهر العسل (+ إضافة باقة)، الصفحة الرئيسية، المدونة (+ إضافة/تعديل مقالة)، القائمة العلوية، الموبايل، الإعدادات، الذكاء الاصطناعي، بذر البرامج

### الصفحات التي تحتاج إصلاح (4 صفحات)

| الصفحة | المشكلة |
|--------|---------|
| **المقالات** (`/admin/articles`) | صفحة مكررة مع المدونة، تستخدم بيانات وهمية فقط (mock data) — لا تحفظ في DB |
| **المدفوعات** (`/admin/payments`) | تعمل لكن عمود `status` في الـ query بحاجة لمطابقة (`completed` vs `success`) — عرض ناقص |
| **التقييمات** (`/admin/reviews`) | تعمل لكن لا توجد وظيفة حذف أو رد على التقييم |
| **التقارير** (`/admin/reports`) | `topDestinations` دائماً `bookings: 0` — لا يتم ربطها فعلياً بعدد الحجوزات |

### المشاكل المنطقية بين الصفحات

1. **تكرار: Articles vs Blog** — صفحة `/admin/articles` مكررة مع `/admin/blog` ولا تحفظ في DB
2. **التقارير لا تعرض بيانات حقيقية** للوجهات الأكثر حجزاً
3. **المدفوعات** لا تعرض تفاصيل كافية ولا يمكن تصدير CSV
4. **التقييمات** لا يمكن الرد عليها (عمود `response` موجود في DB لكن غير مستخدم)

---

## خطة الإصلاح

### 1. حذف صفحة المقالات المكررة
- إزالة `/admin/articles` من الـ routes في `App.tsx`
- إزالتها من القائمة الجانبية في `AdminLayout.tsx` (لا يبدو أنها موجودة أصلاً، لكن يوجد route)
- صفحة المدونة `/admin/blog` هي البديل الصحيح والمتصل بـ Supabase

### 2. تحسين صفحة التقارير
- ربط `topDestinations` فعلياً بعدد الحجوزات عبر `bookings.program_id → programs → destinations` أو مباشرة
- إضافة رسم بياني شهري بسيط (bookings per month aggregation)
- تصدير CSV شامل يتضمن التفاصيل

### 3. تحسين صفحة التقييمات
- إضافة زر "رد" مع textarea لحفظ الرد في عمود `response` الموجود في DB
- إضافة زر حذف التقييم
- عرض `reviewable_type` بشكل مفهوم (فندق/برنامج/وجهة)

### 4. تحسين صفحة المدفوعات
- إضافة زر تصدير CSV
- عرض تفاصيل القسط (`installment_plan`) إن وجد
- عرض سبب الفشل (`failure_reason`) في حالة الرفض
